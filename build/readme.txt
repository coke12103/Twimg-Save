# Twimg Save
バージョン: 4.0.1

推奨スペック: Windows 10またはWindows 7が正常に動作する環境の場合快適に動作するものと思われます。

説明: Twimg Saveは主に画像を保存するためのアプリケーションです。基本的には画像のみですが、一部のプラットフォームでは画像以外のファイルも保存することができます。

今現在、対応しているプラットフォームは以下の通りです。
1. Twitter
2. Pixiv
3. Mastodon
4. Misskey
5. Pleroma
6. Danbooru(danbooru.donmai.us)
7. Yandere(yande.re)

これらのプラットフォームからのデータ取得は、プラットフォーム上の用意されたAPIが利用可能な場合それを利用しますが、用意されたAPIが利用不可能、または存在しない場合はクローラーの動作を応用した処理によって実現されます。
前記の仕様上、Pixivの複数枚投稿のダウンロード時のみ、画像1枚辺り0.8~2.5秒の待機時間が発生し、大多数の投稿は0.8秒の待機しか発生しませんが、複数枚の投稿で2枚目以降の画像形式が違う場合のみ1~2.5秒の待機が発生します。

ダウンロードされたファイルは同一の命名規則に従って命名され保存されます。命名規則は以下の通りです
{SNSの種類を表す英字2~3文字}_{投稿者のID}(_{ユーザーのドメイン})_(投稿のID)_image{何枚目か}.{拡張子}
括弧の中のユーザーのドメインは設定で有効した場合でプラットフォームが分散SNSの場合でのみ付与されます。

ダウンロードされたファイルの保存先はカテゴリによってコントロール可能です。カテゴリは名前と保存先の2つの要素を持ち、任意に追加、削除、編集が可能です。カテゴリはcategorys.jsonというファイル名で実行ファイルと同じ場所に保存されます。

Twimg Saveにはクリップボードを監視し、対応プラットフォームのURLが検知されるとURLをフォームに入れる機能があります。この機能は監視のみをONにした場合はURLをフォームに入れるだけですが、この時自動保存モードも同時にONにすると、ダウンロードの実行も同時に行なわれます。

設定はすべての設定がGUI上から変更可能です。

設定項目[categorys.jsonの場所]は通常、実行ファイルと同じ場所に生成されるカテゴリの格納されたファイルの場所を変更できる機能です。以下の手順でバージョンアップ時にカテゴリを引き継ぐことができます。
1. 旧バージョンのフォルダにあるcategorys.jsonを任意の場所に移動する
2. 新バージョンを起動して設定を開き、[categorys.jsonの場所]に手順1で移動したcategorys.jsonの場所を指定する。
3. カテゴリが旧バージョンから引き継げれば成功。
4. 以降のバージョンアップは手順2,3のみで引き継ぐことができます。

設定項目[通知設定]は任意のデスクトップ通知のON/OFFを細かく切り替えることができます。
